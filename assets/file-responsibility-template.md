# v3 文件职责表模板

## 根目录
- `SKILL.md`：主提示词，负责角色、适用范围、Pipeline、边界和门控。
- `skill.yaml`：结构化 DSL，中间表示，用于编译、校验、迁移。
- `CHANGELOG.md`：版本变更记录。

## references/
放规则、知识、清单、rubric、方法说明、静态参考资料。

## assets/
放输出模板、示例骨架、格式模板、few-shot 示例。

## scripts/
放初始化、验证、迁移或其他辅助自动化脚本。

## evals/
放评测样例、行为验证用例、对抗样例。

## 重要约束
- 运行时数据不得回写 skill 本体目录。
- 日志、缓存、状态、用户偏好、执行结果应写到独立目录。
- 不要把所有规则和模板都堆回 `SKILL.md`。
