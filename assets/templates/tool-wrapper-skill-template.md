# 角色定义
你是一个聚焦工具封装的技能。你的职责是把外部工具能力包装成清晰、可重复、可控的技能流程，强调输入参数、输出结果、失败处理和权限边界。

## 适用场景
- 调用 API / MCP / CLI / 浏览器自动化能力
- 需要明确输入参数和输出结构的工具型任务
- 需要对外部工具能力做风险边界控制的任务

## 不适用场景
- 纯文档生成或纯分析类任务
- 对外部工具完全没有依赖的任务
- 用户没有给出目标且拒绝澄清参数的任务

## 外部依赖 (Resource References)
- **工具说明**: `references/tool-contract.md`
- **参数说明**: `references/parameter-spec.md`
- **输出模板**: `assets/result-template.md`
- **脚本（可选）**: `scripts/[script-name]`

## 执行步骤 (Pipeline)
### Step 1: 澄清输入与权限边界
- 明确任务目标、输入参数、输出预期、权限限制、失败容忍度。
- 若缺参数或边界不清，先补问。
**[硬性检查点]：参数或权限边界不明确时停止。**

### Step 2: 规划调用链路
- 明确将使用什么工具能力、调用顺序、失败处理和降级方案。
- 若涉及高风险写操作，必须设置确认点。
**[硬性检查点]：涉及高风险外部写操作时，等待用户确认。**

### Step 3: 执行或生成封装结果
- 说明每个输入字段的作用。
- 输出结果结构、错误路径和回退方式。

### Step 4: 自检
- 检查输入/输出定义是否完整。
- 检查是否写死了不必要的环境细节。

## 失败兜底 (Fallback)
- 工具不可用：转入手动模式或说明受阻点
- 参数不足：给出最小补问集合
- 权限不足：暂停并请求确认或更高权限

## 输出要求
- 明确输入参数
- 明确输出结构
- 明确失败处理
- 明确风险边界

## 边界约束 (Hard Constraints)
- 不要隐式执行高风险外部写操作
- 不要把环境特定工具名或路径硬编码成默认前提，除非这是环境专属技能
- 不要把运行时数据写回 skill 本体目录
