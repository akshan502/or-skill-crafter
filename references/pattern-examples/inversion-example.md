# Inversion 模式完整示例

本文档展示 Inversion 模式的完整可运行示例：项目规划面试官。

---

## 技能结构

```
skills/project-planner/
├── SKILL.md                      # 多阶段面试流程
└── assets/
    └── plan-template.md          # 最终输出模板
```

---

## SKILL.md

```markdown
---
name: project-planner
description: Plans new software projects by gathering requirements through structured interviews before producing a plan. Use when user says "I want to build", "help me plan", "design a system", "start a new project", "create an architecture". Trigger keywords: plan, project, build, design system, architecture, new project.
metadata:
  pattern: inversion
  interaction: multi-turn
  phases: 3
  depends_on:
    assets:
      - plan-template.md
---

# Project Planner

You are conducting a structured requirements interview for a software project.

## Critical Gate

**DO NOT start building, designing, or writing code until ALL phases are complete.**

Your role is to ASK first, then SYNTHESIZE after gathering complete information.

---

## Phase 1 — Problem Discovery

Ask these questions **ONE AT A TIME**. Wait for each answer before asking the next.

### Q1
"What problem does this project solve for its users?"

Wait for user answer. Acknowledge briefly, then ask Q2.

### Q2
"Who are the primary users? What is their technical level?"

Wait for user answer. Acknowledge briefly, then ask Q3.

### Q3
"What is the expected scale? (users per day, data volume, request rate)"

Wait for user answer. 

**Phase 1 Complete Gate**: Summarize what you learned about the problem and users. Ask: "Is this accurate? Should I proceed to technical constraints?"

DO NOT proceed to Phase 2 until user confirms.

---

## Phase 2 — Technical Constraints

**Only start after Phase 1 is confirmed complete.**

Ask these questions **ONE AT A TIME**:

### Q4
"What deployment environment will you use? (cloud, on-premise, hybrid, mobile)"

Wait for answer.

### Q5
"Do you have technology stack requirements or preferences?"

Wait for answer.

### Q6
"What are the non-negotiable requirements? (latency, uptime, compliance, budget)"

Wait for answer.

**Phase 2 Complete Gate**: Summarize technical constraints. Ask: "Have I captured all constraints? Should I proceed to synthesis?"

DO NOT proceed to Phase 3 until user confirms.

---

## Phase 3 — Synthesis

**Only start after ALL phases are confirmed complete.**

1. Load `assets/plan-template.md` for output structure

2. Fill in EVERY section using gathered requirements:
   - Problem Statement → from Q1
   - User Analysis → from Q2
   - Scale Requirements → from Q3
   - Deployment → from Q4
   - Technology Stack → from Q5
   - Non-negotiable Constraints → from Q6

3. Present the completed plan

4. Ask: "Does this plan accurately capture your requirements? What would you change?"

5. Iterate on feedback until user says the plan is satisfactory

---

## Hard Constraints

- NEVER ask multiple questions at once (batching)
- NEVER proceed to next phase without user confirmation
- NEVER synthesize before all 6 questions answered
- NEVER write code or design architecture during interview phases
- NEVER skip any question (even if user seems impatient)
```

## 使用示例

### 用户触发

```
用户: "我想做一个在线教育平台"
```

### Agent 行为（Phase 1）

Agent: "What problem does this project solve for its users?"

用户: "帮助学生找到合适的老师，一对一在线辅导"

Agent: "了解。Who are the primary users? What is their technical level?"

用户: "主要是高中生和大学生，技术能力一般"

Agent: "好的。What is the expected scale? (users per day, data volume, request rate)"

用户: "初期大概1000用户每天，后面希望增长到10000"

Agent: 
```
Phase 1 总结：
- 问题：帮助学生匹配合适老师，一对一辅导
- 用户：高中生和大学生，技术能力一般
- 规模：1000 → 10000 用户/天

Is this accurate? Should I proceed to technical constraints?
```

用户: "是的，继续"

### Agent 行为（Phase 2）

Agent: "What deployment environment will you use? (cloud, on-premise, hybrid, mobile)"

用户: "云端部署，国内服务器"

...

### 最终输出

Phase 3 加载模板，填充所有字段，输出完整项目计划。

## 模式合规检查点

| 检查项 | 要求 |
|--------|------|
| 门控指令 | 包含 "DO NOT...until..." 约束 |
| 单一提问 | 问题逐个询问，不批量 |
| 阶段划分 | Phase 1/2/3 明确定义 |
| 阶段间门控 | 每个阶段结束要求用户确认 |
| 延迟合成 | Phase 3 仅在所有阶段完成后执行 |
